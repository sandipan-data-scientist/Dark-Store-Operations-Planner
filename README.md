Prophet + LightGBM · FastAPI · Streamlit · HuggingFace Spaces · Docker

## 🚀 Quick Start

### 1. Install Dependencies

```bash
pip install -r requirements.txt
python -c "import cmdstanpy; cmdstanpy.install_cmdstan()"
```

### 2. Place the Dataset

```bash
cp delhi_ncr_darkstore_fruit_vegetable_sales_2022_2025.csv data/
```

### 3. Train & Pickle Models (required before first run)

```bash
# Full training — all 32 products (~10-20 min)
python scripts/train_and_pickle.py

# Quick demo — top 5 products (~2-3 min)
python scripts/train_and_pickle.py --fast
```

This generates the `models/` directory with LightGBM `.txt` files, Prophet forecast `.pkl` files, and analytics artifacts.

### 4. Start the App Locally

```bash
bash start.sh
# FastAPI docs: http://localhost:8000/docs
# Streamlit UI: http://localhost:7860
```

## 🐳 Deploy to HuggingFace Spaces

### Option A: Docker Space (Recommended)

1. Create a new HuggingFace Space → **Docker** type  
2. Commit this project (including `models/` after training):

```bash
git init
git remote add hf https://huggingface.co/spaces/YOUR_USERNAME/darkstore-planner
git add .
git commit -m "initial deployment"
git push hf main
```

> **Note**: If `models/` directory is large, use [Git LFS](https://git-lfs.com):  
> `git lfs track "models/*.pkl" "models/*.txt"`

3. HuggingFace builds the Docker image (includes model training if `models/` not committed)  
4. The Space runs Streamlit on port 7860 with FastAPI on 8000 internally

### Option B: Pre-bake models in Docker image

```bash
# The Dockerfile runs train_and_pickle.py during build by default
# This takes ~10-15 min but results in a self-contained image
docker build -t darkstore-planner .
docker run -p 7860:7860 darkstore-planner
```

### Option C: Skip build-time training (commit models instead)

```bash
docker build --build-arg SKIP_PRETRAIN=1 -t darkstore-planner .
```

## 📊 Application Sections

### 1. 📓 Notebook Findings & Inferences
- Full EDA reproduction with interactive Plotly charts
- Daily sales trends, YoY overlay, day-of-week patterns
- Seasonal index heatmaps per product × month
- Walk-forward validation metrics (MAPE, MAE, RMSE per product)
- Seasonal cross-year deviation analysis

### 2. 📈 Analytics Dashboard
- Custom date range selector with previous period comparison
- KPI cards: total sales, daily average, peak day, zero-sale days
- Product-level time series (daily/weekly/monthly/quarterly)
- Product share pie, monthly heatmap, DoW patterns
- Period-over-period growth table
- Pearson correlation matrix

### 3. 🏭 Supply Chain & Inventory Planning
- **Liability cost inputs**: transport, last mile, manpower, storage & electricity, rent & licences, miscellaneous
- **Product expiry timeline configuration** (shelf life per product in days)
- **Dynamic vendor pricing table** (3+ vendors; blank = not available)
- **Weekly restocking plan**: recommended order qty, safety stock, degradation risk, stockout risk
- **Risk metrics**: amount at risk, degradation loss value, net profit estimate
- **P&L waterfall chart**
- **Daily demand forecast + confidence interval + CV risk gauge**

### 4. 💰 Pricing Dashboard
- **Per-product unit configuration**: kg / grams / units (with average unit weight)
- **3-column pricing table**: your price · competitor price · vendor cost
- Automatic per-kg equivalent calculation for unit-based products
- **Competitive positioning**: % above/below competitor per product
- **Margin analysis**: gross margin, margin per kg, color-coded heatmap
- **Monthly P&L forecast**: revenue, profit, deviation from past 3-month average
- **Optimal price range**: floor (target margin) → suggested → ceiling (5% above competitor)
- Download pricing report as CSV

## 🤖 Models

| Model | Use Case | Horizon | Features |
|---|---|---|---|
| LightGBM | Daily / weekly demand | 1–30 days | lag-7/14/30/60/365, rolling mean/std, Fourier, calendar |
| Prophet | Monthly / quarterly / annual | 30–365 days | Multiplicative seasonality, yearly + weekly components |

### Performance Benchmarks
- **Staples** (Potato, Onion, Tomato): MAPE 8–12%
- **Year-round fruits** (Banana, Apple): MAPE 10–15%
- **Seasonal fruits** (Mango, Watermelon): MAPE 18–30% in-season

## ⚙️ Configuration

Edit `app/config.py` to adjust:
- `DEFAULT_SHELF_LIFE`: shelf-life defaults per product
- `PROPHET_PARAMS`: Prophet model hyperparameters
- `LGBM_PARAMS`: LightGBM hyperparameters
- `LAG_FEATURES`: lag offsets for feature engineering
- `FORECAST_HORIZON_DAYS`: how far ahead Prophet forecasts (default 365)

## 🔧 Troubleshooting

| Issue | Solution |
|---|---|
| `models not found` | Run `python scripts/train_and_pickle.py` first |
| CmdStan install fails | Run `python -c "import cmdstanpy; cmdstanpy.install_cmdstan()"` |
| HF Spaces build timeout | Use `--build-arg SKIP_PRETRAIN=1` and commit pre-trained models |
| Large model files on HF | Use `git lfs track "models/*.pkl"` before committing |
| Port 7860 in use locally | `export PORT_APP=8501 && bash start.sh` |
